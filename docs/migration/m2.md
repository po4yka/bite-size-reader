  - `rust/crates/bsr-summary-contract/tests/m2_fixture_parity.rs`

## Backend Selection

`SUMMARY_CONTRACT_BACKEND` controls runtime behavior:

- `python` (default): always use Python contract implementation.
- `auto`: use Rust when binary is present, fallback to Python.
- `rust`: prefer Rust and fallback to Python if unavailable/failing.

Optional binary override:

- `SUMMARY_CONTRACT_RUST_BIN=/abs/path/to/bsr-summary-contract`

## SQLite Compatibility Scope

M2 validates Rust compatibility against the existing SQLite contract shape by:

- Schema checks for required core columns in `requests` and `summaries`.
- Transactional write/read smoke test (`sqlite_roundtrip_smoke`) with rollback.
- Snapshot compatibility test against `app_backup.db`.
