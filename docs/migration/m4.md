## Backend Selection and Rollback

`MIGRATION_INTERFACE_BACKEND` controls interface-layer routing behavior:

- `python` (default): Python-only route/command resolver.
- `canary`: sampled traffic resolved by Rust (`MIGRATION_INTERFACE_SAMPLE_RATE`), Python fallback on errors.
- `rust`: always prefer Rust resolver, Python fallback on errors.

Related controls:

- `MIGRATION_INTERFACE_SAMPLE_RATE` (default `0.0`)
- `MIGRATION_INTERFACE_TIMEOUT_MS` (default `150`)
- `MIGRATION_INTERFACE_EMIT_MATCH_LOGS` (default `false`)
- `MIGRATION_INTERFACE_MAX_DIFFS` (default `8`)
- `INTERFACE_ROUTER_RUST_BIN` (optional binary path override)

This provides an operational rollback switch by setting `MIGRATION_INTERFACE_BACKEND=python`.
