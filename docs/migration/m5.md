## Runtime Defaults and Rollback

M5 defaults production traffic to Rust-first paths while preserving rollback toggles:

- `SUMMARY_CONTRACT_BACKEND=rust` (rollback: `python`)
- `MIGRATION_INTERFACE_BACKEND=rust` (rollback: `python`)

Python fallback behavior remains enabled during the release window and is emitted as standardized cutover events.

## Release-Window Verification

Use the M5 checker to verify no Python fallback usage during the release window:

```bash
PYTHONPATH=. .venv/bin/python scripts/migration/check_m5_cutover_window.py \
  --events-file /data/migration_cutover_events.jsonl \
  --window-days 14
```

Exit status:

- `0`: no fallback events in the window
- `1`: fallback events detected
- `2`: events file missing (unless `--allow-missing-file` is set)
