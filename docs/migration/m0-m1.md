# M0–M1 Migration Implementation Notes

This document tracks concrete repository artifacts delivered for migration milestones M0 and M1.

## M0 — Baseline and Contract Freeze

Implemented artifacts:

- Parity suite runner: `scripts/migration/run_parity_suite.sh`
- Parity tests: `tests/parity/test_migration_parity_contracts.py`
- Baseline metrics capture: `scripts/migration/capture_python_baseline.py`
  - captures `git_commit` for traceable run-to-code mapping
- Recorded baseline output: `docs/migration/baseline_metrics.json`
- Baseline history log: `docs/migration/baseline_metrics_history.jsonl`
- Make targets: `make parity-suite`, `make capture-baseline`

Contract surfaces covered by the parity suite:

- Telegram command alias routing semantics
- Mobile API request defaults and response envelope shape
- Summary JSON normalization and required enrichment fields

## M1 — Rust Foundation (Non-Critical Path)

Implemented artifacts:

- Rust workspace root: `rust/Cargo.toml`
- Shared crates:
  - `rust/crates/bsr-config` (runtime config loading from env)
  - `rust/crates/bsr-logging` (structured JSON logging bootstrap with safe fallback for invalid log levels)
  - `rust/crates/bsr-summary-contract` (summary schema validation helpers)
- CI jobs in `.github/workflows/ci.yml`:
  - `parity-suite` (M0 contract suite)
  - `rust-foundation` (cargo check/test)
- Make targets: `make rust-check`, `make rust-test`

At the M1 stage, these additions kept Python as the production orchestrator while establishing Rust-first shared primitives for later milestones.

For milestone M2 implementation details, see [`docs/migration/m2.md`](m2.md).
