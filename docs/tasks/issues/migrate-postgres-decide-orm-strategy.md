- [x] #task Decide ORM strategy for Postgres migration #repo/ratatoskr #area/db #status/done 🔺 ✅ 2026-05-06

## Objective

Confirm whether the migration stays on Peewee + `playhouse.postgres_ext` (recommended)
or simultaneously moves to SQLAlchemy 2.0 + asyncpg.

## Context

Current state:

- 49 Peewee models across `app/db/_models_*.py`.
- Async wrapper is `asyncio.to_thread(...)` in
  `app/db/runtime/operation_executor.py` — not peewee-async.
- Alembic is already wired (`app/db/alembic/`), uses SQLAlchemy under the hood for
  schema management but the runtime ORM is still Peewee.

Plan recommendation: stay on Peewee. Driver swap is a localised change to
`app/db/session.py`; Postgres equivalents (`PostgresqlExtDatabase`, `BinaryJSONField`,
`TSVectorField`, `PooledPostgresqlExtDatabase`) all exist. A SQLAlchemy port can happen
later as a separate effort.

Pursuing SQLAlchemy in the same pass would roughly triple the diff and overlap with
in-flight feature work; absent a strong async-Postgres performance need, it does not
pay for itself here.

## Acceptance criteria

- [ ] Owner records "stay on Peewee" or "migrate to SQLAlchemy 2.0" in this issue.
- [ ] If SQLAlchemy: this migration plan is reframed as a two-stage effort and a
      follow-up planning task is filed.
- [ ] If Peewee: this issue is closed and downstream tasks unblock.

## Decision

**Port to SQLAlchemy 2.0 + asyncpg simultaneously with the Postgres swap** — confirmed
2026-05-06.

This is a substantially larger effort than the recommended Peewee-stay path:

- **All 49 Peewee model classes** in `app/db/_models_*.py` are rewritten in
  SQLAlchemy 2.0 typed-`Mapped[T]` style with `DeclarativeBase`.
- **Async-native sessions**: `create_async_engine("postgresql+asyncpg://…")` +
  `async_sessionmaker[AsyncSession]`.
- **`asyncio.to_thread` wrappers and `AsyncRWLock` are deleted** — they exist only
  to compensate for SQLite's single-writer semantics. Postgres MVCC removes the
  need entirely.
- **Every call site** that uses `Model.select().where(...)`, `.get()`, `.create()`,
  `.update()`, `.execute()` is rewritten to `select(Model).where(...)` and
  `await session.execute(...)`.
- **Repositories under `app/infrastructure/persistence/`** are rewritten against
  `AsyncSession`.
- **Alembic** stays, now driven by SQLAlchemy autogenerate against the new
  declarative metadata; the historical SQLite-flavoured revisions are kept as
  archived references but the new baseline is autogenerated post-port.
- **Peewee dependency is removed** in the cleanup phase.

Implication: the migration plan now runs through phase **M** (model layer port) and
phase **R** (repository / call-site port) in addition to the original F/O/T/C/L
phases. See `docs/tasks/migrate-sqlite-to-postgresql-plan.md` for the revised task
graph.
